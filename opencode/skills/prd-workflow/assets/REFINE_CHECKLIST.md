# Request Refinement Checklist

A request is ready for PRD conversion when:

1. Problem is explicit (current state, pain, why now)
2. Users are concrete (primary + secondary)
3. Goals are measurable (or clearly directional)
4. Non-goals are explicit
5. Flow is clear (entry, main, edge states)
6. Requirements are testable (FR/NFR)
7. Metrics include targets and timeframe/date
8. Risks include mitigation ideas
9. Open questions capture unresolved conflicts

Refinement output format:
- Strengths (keep)
- Gaps (fix now)
- Suggested edits by section (copy-paste)
- Ready verdict: READY | NOT_READY
